- **Base Lifecycle & DI**: `base_component.py` defines `BaseComponent`, an abstract class implementing an async lifecycle (`start`, `close`, `restart`) with thread-safe locking. It features a custom dependency injection system via `bind()`, which creates `Dependency` placeholders resolved at startup either from a global `ApplicationContext` or via local factories in standalone mode.
- **Global Registry**: `component_registry.py` exposes a singleton `ComponentRegistry` (`R`) that maps `(ComponentEnum, name)` pairs to component classes, supporting both direct registration and decorator-based syntax (`@R.register`).
- **Context Management**: `application_context.py` acts as a passive state container holding the parsed `ApplicationConfig`, wired components, jobs, and service instances. `runtime_context.py` provides a per-request scratch space for streaming responses and transient data.
- **Prompt Utilities**: `prompt_handler.py` offers a specialized utility for loading and rendering prompt templates from YAML/JSON files, featuring i18n support (language-suffixed keys) and conditional line filtering via `[flag]` tags.
