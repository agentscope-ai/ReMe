- **Layered Structure**: The module separates concerns into public Step classes (e.g., `ReadStep`, `WriteStep`) registered via `@R.register`, shared utilities in underscored modules (`_file_io.py` for async I/O/locking, `_path.py` for resolution/validation), and domain-specific helpers (`_daily_index.py`).
- **Concurrency Control**: Uses an in-process `asyncio.Lock` registry (`get_path_lock`) to serialize read-modify-write cycles on a per-path basis, preventing race conditions during concurrent step execution.
- **Security Boundary**: All path inputs are processed through `resolve_path`, which enforces workspace containment (preventing directory traversal) and validates filename components against OS-reserved names and invalid characters.
- **Markdown Specialization**: Leverages the `python-frontmatter` library to distinguish between YAML metadata and body content, enabling safe frontmatter CRUD (`FrontmatterReadStep`, `FrontmatterUpdateStep`) and body-only edits (`EditStep`) that preserve structural integrity.
