- **Declarative Wiring**: The `Application` class acts as the central orchestrator, parsing `default.yaml` or `demo.yaml` to instantiate components, jobs, and services via the global `ComponentRegistry` (`R`).
- **Shared Runtime Context**: An `ApplicationContext` singleton holds the typed `ApplicationConfig`, wired component instances, and active jobs, serving as the primary dependency injection container for both long-lived components and ephemeral execution steps.
- **Dual Dependency Resolution**: Long-lived `BaseComponent` instances use async binding resolution at startup (supporting topological ordering), while ephemeral `BaseStep` units use lazy `Ref` descriptors to resolve dependencies from the context or call-time kwargs during job execution.
- **Lifecycle Management**: The framework enforces a strict async lifecycle (`start`/`close`) with reverse-order shutdown for components and categorized execution for jobs (base, stream, background, cron), ensuring resource safety and ordered teardown.
