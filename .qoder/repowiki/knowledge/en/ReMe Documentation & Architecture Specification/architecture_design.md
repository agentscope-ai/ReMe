- **Layered Runtime Model**: Defines a six-layer execution stack (L0-L5) where L0 is the workspace filesystem, L1 is the file watcher, L2 is the file state/index, L3 contains atomic tools, L4 orchestrates Jobs/Steps, and L5 exposes Services/Runtime.
- **Memory-as-File Paradigm**: Structures memory into four workspace layers: `session/` (raw input), `resource/` (external materials), `daily/` (working memory), and `digest/` (long-term knowledge), all persisted as Markdown with frontmatter and wikilinks.
- **Component-Step-Job Hierarchy**: Describes the internal assembly of `Application`, where `Jobs` orchestrate atomic `Steps` that interact with reusable `Components` (e.g., `file_store`, `agent_wrapper`) via a global registry and dependency injection.
- **Dual-Process Design**: Distinguishes between the `Service` process (handling synchronous agent requests like ingest/retrieve) and the `Runtime` process (managing asynchronous background tasks like auto-dreaming and maintenance).
